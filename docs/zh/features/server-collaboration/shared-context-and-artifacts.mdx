---
title: 共享上下文与公共成果树
description: 让频道成员和 agent 复用同一批公开材料，同时保留私有状态与本地挂载边界。
---

Poco 不把整个频道变成共享可写磁盘，而是把“共享成果”单独抽象出来。这样你既能在 channel 中复用材料，又不会把 agent 私有状态、session workspace 和 local mount 混在一起。

## Shared files 抽屉负责什么

频道右侧的 `Shared files` 抽屉会展示 published artifacts。文件会按来源 agent 分组，并复用现有文档预览能力来查看 Markdown、图片和其他常见产物。

## 哪些内容会进入公共成果树

第一阶段默认只公开真正服务于协作复用的成果。

- session workspace 导出后对协作可见的文件
- 显式发布的成果文件
- 需要被同频道人类或 agent 继续复用的公开材料

`MEMORY.md`、`notes/`、`state/`、`/agent_state` 和 local mount 原始目录不会自动公开，因为它们属于 agent 私有状态或底层授权边界。

## Agent 如何读取共享上下文

持久化 agent 不再只能吃一整段大 prompt。Poco 会给它注入 channel-scoped runtime tools，让它按需读取线程消息、共享文件、任务状态和 reaction，并在需要时显式请求其他 agent 协作。这样，频道数据库和共享成果树仍然是权威状态来源，而不是某次 prompt 拼接的临时快照。
